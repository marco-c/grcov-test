      95             : // A zone is a collection of compartments. Every compartment belongs to exactly
      96             : // one zone. In Firefox, there is roughly one zone per tab along with a system
      97             : // zone for everything else. Zones mainly serve as boundaries for garbage
      98             : // collection. Unlike compartments, they have no special security properties.
      99             : //
     100             : // Every GC thing belongs to exactly one zone. GC things from the same zone but
     101             : // different compartments can share an arena (4k page). GC things from different
     102             : // zones cannot be stored in the same arena. The garbage collector is capable of
     103             : // collecting one zone at a time; it cannot collect at the granularity of
     104             : // compartments.
     105             : //
     106             : // GC things are tied to zones and compartments as follows:
     107             : //
     108             : // - JSObjects belong to a compartment and cannot be shared between
     109             : //   compartments. If an object needs to point to a JSObject in a different
     110             : //   compartment, regardless of zone, it must go through a cross-compartment
     111             : //   wrapper. Each compartment keeps track of its outgoing wrappers in a table.
     112             : //   JSObjects find their compartment via their ObjectGroup.
     113             : //
     114             : // - JSStrings do not belong to any particular compartment, but they do belong
     115             : //   to a zone. Thus, two different compartments in the same zone can point to a
     116             : //   JSString. When a string needs to be wrapped, we copy it if it's in a
     117             : //   different zone and do nothing if it's in the same zone. Thus, transferring
     118             : //   strings within a zone is very efficient.
     119             : //
     120             : // - Shapes and base shapes belong to a zone and are shared between compartments
     121             : //   in that zone where possible. Accessor shapes store getter and setter
     122             : //   JSObjects which belong to a single compartment, so these shapes and all
     123             : //   their descendants can't be shared with other compartments.
     124             : //
     125             : // - Scripts are also compartment-local and cannot be shared. A script points to
     126             : //   its compartment.
     127             : //
     128             : // - ObjectGroup and JitCode objects belong to a compartment and cannot be
     129             : //   shared. There is no mechanism to obtain the compartment from a JitCode
     130             : //   object.
     131             : //
     132             : // A zone remains alive as long as any GC things in the zone are alive. A
     133             : // compartment remains alive as long as any JSObjects, scripts, shapes, or base
     134             : // shapes within it are alive.
