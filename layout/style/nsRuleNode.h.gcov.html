     347             :  * indexed by style rules (implementations of nsIStyleRule).
     348             :  *
     349             :  * The rule tree is owned by the nsStyleSet and is destroyed when the
     350             :  * presentation of the document goes away. Its entries are reference-
     351             :  * counted, with strong references held by child nodes, style structs
     352             :  * and (for the root), the style set. Rule nodes are not immediately
     353             :  * destroyed when their reference-count drops to zero, but are instead
     354             :  * destroyed during a GC sweep.
     355             :  *
     356             :  * An nsStyleContext, which represents the computed style data for an
     357             :  * element, points to an nsRuleNode.  The path from the root of the rule
     358             :  * tree to the nsStyleContext's mRuleNode gives the list of the rules
     359             :  * matched, from least important in the cascading order to most
     360             :  * important in the cascading order.
     361             :  *
     362             :  * The reason for using a lexicographic tree is that it allows for
     363             :  * sharing of style data, which saves both memory (for storing the
     364             :  * computed style data) and time (for computing them).  This sharing
     365             :  * depends on the computed style data being stored in structs (nsStyle*)

     367             :  * ("inherited structs") or structs that contain only properties that
     368             :  * are not inherited by default ("reset structs").  The optimization
     369             :  * depends on the normal case being that style rules specify relatively
     370             :  * few properties and even that elements generally have relatively few
     371             :  * properties specified.  This allows sharing in the following ways:
     372             :  *   1. [mainly reset structs] When a style data struct will contain the
     373             :  *      same computed value for any elements that match the same set of
     374             :  *      rules (common for reset structs), it can be stored on the
     375             :  *      nsRuleNode instead of on the nsStyleContext.
